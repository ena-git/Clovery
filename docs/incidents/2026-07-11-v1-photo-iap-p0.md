# V1 照片保存与内购权益 P0 处理记录

**状态：** 客户端修复完成，等待 App Store Connect 商品核验与真机生产验证
**热修复版本：** `1.0.2 (13)`
**影响范围：** 今日留言板保存到系统相册、永久解锁购买与恢复购买

## 已确认根因

### 保存到相册

旧实现调用 `UIImageWriteToSavedPhotosAlbum` 后立即通知网页“保存成功”，没有等待系统照片库写入完成，也没有把权限拒绝和写入错误返回给界面。因此用户可能看到保存状态，但相册中没有图片；权限被拒绝时则完全没有反馈。

修复后使用 `.addOnly` 权限与 `PHPhotoLibrary.performChanges` 创建照片资产，只在 Photos completion handler 返回成功后显示“已保存”。权限拒绝、无效图片和写入失败均返回明确状态，中文界面提示用户前往系统设置授权或重试。

### 购买后未解锁

旧 Swift→JavaScript 回调把 Swift 字符串插值转义成了字面量，网页无法收到真实的 `success`、布尔解锁值和本地化价格。旧 `purchase()` 只返回 Bool，无法区分取消、待批准和失败；TestFlight 又被无条件视为已解锁，导致测试无法覆盖真实购买链路。

修复后：

- JS 参数统一通过 JSON 编码生成，禁止字符串拼接和双反斜杠插值。
- 购买状态固定为 `success`、`cancelled`、`pending`、`failed`。
- 仅在 StoreKit 验证交易成功后解锁并 `finish()`。
- 启动时读取 `Transaction.currentEntitlements`，并监听 `Transaction.updates` 处理 Ask to Buy 等延迟交易。
- 移除 TestFlight 自动解锁；测试环境也必须完成真实 Sandbox 交易或恢复。
- 商品缺失、验证失败、购买失败和恢复失败写入 StoreKit 日志，不再静默吞掉。

## App Store Connect 上线闸门

截图中的“App 内购买项目 0”属于趋势指标，不等同于商品配置页面。发布 `1.0.2` 前必须在 App Store Connect 的 App → Monetization → In-App Purchases 中逐项确认：

1. 存在商品 ID `com.clovery.app.board.lifetime`，类型为 Non-Consumable。
2. 商品属于 Bundle ID `com.clovery.app` 对应的 Clovery App。
3. 已填写参考名称、中文/英文显示名称、描述、审核截图和价格。
4. 中国大陆及计划运营地区处于可售状态。
5. Paid Apps Agreement、税务和收款资料均有效。
6. 商品状态达到 Ready to Submit；首次内购必须与新的 App 版本 `1.0.2` 一并提交审核。
7. App 版本页面的 In-App Purchases and Subscriptions 区域已选择该商品。
8. 保存配置后至少等待 Sandbox 元数据同步，再使用 Sandbox Apple Account 完成购买、取消、待批准和恢复测试。

若 `Product.products(for:)` 返回空数组，客户端会显示购买失败并记录 `Product unavailable`；这通常表示商品 ID、商品状态、协议、地区或审核绑定仍有一项不满足。

## 已反馈用户处理

1. 只向用户索取 App Store 订单/交易凭证截图、购买时间、App 版本和 iOS 版本；绝不索取 Apple ID 密码或验证码。
2. 若 Apple 购买历史中存在 `com.clovery.app.board.lifetime` 对应交易，用户升级 `1.0.2` 后点击“已购买？恢复购买”，StoreKit 当前权益应重新解锁。
3. 若购买历史和 App Store Connect 交易均不存在，先确认是否只是银行卡预授权或 Sandbox/TestFlight 交易，再由 Apple 处理扣款查询或退款。
4. V1 没有 Clovery 服务端权益账本，不能根据聊天截图安全地跨设备人工授予永久权益。V2 必须把验证后的交易与 `clovery_account_id` 绑定。

## 验收命令

```bash
./scripts/test-v1-bridge.sh
./scripts/test-v1-p0-contract.sh
node scripts/validate-v1-html.cjs
xcodebuild -quiet -project Clovery.xcodeproj -scheme Clovery -configuration Debug -sdk iphonesimulator CODE_SIGNING_ALLOWED=NO build
```

真机发布前还必须完成：首次授权保存、已授权保存、拒绝权限、购买成功、取消、Ask to Buy pending、恢复购买、卸载重装后恢复，以及生产 App Store 商品可见性测试。
