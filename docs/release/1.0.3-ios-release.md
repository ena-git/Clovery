# Clovery iOS 1.0.3 Release Candidate

**Release type:** Existing App Store application upgrade
**Candidate version:** `1.0.3 (14)`
**Production baseline:** `1.0.2 (13)`
**Git branch:** `codex/w0-platform-foundation`
**Planned tag:** `ios-v1.0.3`

原生 iOS 1.0.3 (14) 的自动化、升级数据、相册、迁移、StoreKit、TestFlight 和 App Store 发布门禁必须在 Flutter W3 开始前完成。旧文档中 W3/W4 后再完成 W1 真机验收的排期由 2026-07-15 已批准方案 A 替代。

## Scope

- Preserve the existing bundle IDs, App Group, iCloud container, user data, and
  App Store listing.
- Include the V1 photo-library save fix and StoreKit entitlement recovery fix.
- Include migration export support required by the later Clovery-account/Vault
  transition.
- Keep the Go backend and Flutter multi-platform work in the repository without
  using the Flutter app as this App Store binary.

## Release Flow

1. Merge the release pull request into `main` without rewriting the existing
   repository history.
2. Build the native `Clovery` Xcode scheme as `1.0.3 (14)`.
3. Complete the iOS real-device, Sandbox, TestFlight, data-retention,
   photo-library, and purchase-restoration gates.
4. Upload build `14` as an upgrade of bundle ID `com.clovery.app`.
5. Submit the existing non-consumable product
   `com.clovery.app.board.lifetime` with the App version when App Store Connect
   requires first-IAP review.
6. Create tag `ios-v1.0.3` and publish the GitHub release only after the shipped
   commit and App Store build are known.

## Blocking Gates

- [ ] Main app and widget both report `1.0.3 (14)`.
- [ ] Simulator build and automated V1 tests pass.
- [ ] Existing local diary, photo, tombstone, and entitlement data survive the
  upgrade installation.
- [ ] Real-device save-to-library succeeds and permission denial is recoverable.
- [ ] Sandbox purchase, cancellation, pending, restore, reinstall, and second
  device cases pass.
- [ ] App Store Connect product metadata and sale territories are complete.
- [ ] TestFlight build passes the release smoke checklist.
- [ ] Release commit is merged to `main` before tagging.

Do not publish the GitHub release or submit the App Store version while any
blocking gate remains open.
